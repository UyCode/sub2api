## Endpoint

```http
POST https://YOUR_DOMAIN/v1/images/generations
```

## 请求示例

<CodeGroup>

```bash cURL
curl -X POST "https://YOUR_DOMAIN/v1/images/generations" \
  -H "Authorization: Bearer $SUB2API_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "A premium studio photo of a translucent mechanical keyboard, crisp reflections, soft teal background.",
    "size": "2048x1152",
    "quality": "medium",
    "output_format": "png",
    "response_format": "url"
  }'
```

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="SUB2API_API_KEY",
    base_url="https://YOUR_DOMAIN/v1",
)

response = client.images.generate(
    model="gpt-image-2",
    prompt="A premium studio photo of a translucent mechanical keyboard.",
    size="2048x1152",
    quality="medium",
    response_format="url",
)

image_url = response.data[0].url
print(image_url)
```

</CodeGroup>

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `model` | string | 否 | `gpt-image-2` | 图片模型 ID。图片接口只接受 `gpt-image-*` 前缀模型。 |
| `prompt` | string | 是 | - | 生成图片的文本描述。建议包含主体、构图、风格、光线、输出用途和限制条件。 |
| `size` | string | 否 | 官方默认 | 输出尺寸，如 `1024x1024`、`2048x1152`、`3840x2160`，会归一到 1K/2K/4K 计费档位。 |
| `quality` | string | 否 | `auto` | 画质档位。生产建议从低档跑通后再提升。 |
| `n` | integer | 否 | `1` | 图片数量。生产建议保持 `1`，多图由业务侧并发或队列控制。 |
| `response_format` | string | 否 | `url` | 推荐 `url`。兼容 OpenAI 的 `b64_json`，但生产链路不建议默认使用。 |
| `output_format` | string | 否 | `png` | 输出格式，如 `png`、`jpeg`、`webp`。 |
| `output_compression` | integer | 否 | - | JPEG/WebP 压缩质量，具体支持取决于官方能力。 |
| `background` | string | 否 | `auto` | 背景策略，如透明背景或自动背景，具体可用值以官方能力为准。 |
| `moderation` | string | 否 | `auto` | 内容审核强度或策略，具体可用值以官方能力为准。 |
| `stream` | boolean | 否 | `false` | 是否启用图片生成流式事件。 |
| `partial_images` | integer | 否 | - | 流式模式下可返回的中间图片数量，需与 `stream` 配合。 |
| `user` | string | 否 | - | 终端用户标识，便于审计、风控和问题定位。 |

<Warning>
  不要在新接入时直接使用高画质 4K 压测。先用 1K/低画质确认鉴权、URL 保存、错误处理和超时，再逐档提高。
</Warning>

## 响应

```json
{
  "created": 1780545600,
  "data": [
    {
      "url": "https://YOUR_DOMAIN/assets/images/req_abc123/output-0.png"
    }
  ]
}
```

DeepToken 会优先返回可下载 URL。URL 形式能显著减少响应体大小：base64 会比原始二进制图片大约 33%，在大图和高并发场景下会拖慢响应并占用更多中转带宽。

<Note>
  如果历史客户端必须处理 base64，可以请求 `response_format: "b64_json"` 并读取 `data[].b64_json`。新系统建议直接使用 `data[].url` 并尽快转存到你的业务对象存储。
</Note>

| 请求尺寸 | 记录档位 | 当前价格 | 说明 |
| --- | --- | ---: | --- |
| `1024x1024` | `1K` | `$0.060/次` | 常规预览、社媒图、快速验证 |
| `2048x1152` | `2K` | `$0.090/次` | 高清横版物料、网站主视觉 |
| `3840x2160` | `4K` | `$0.150/次` | 商业海报、高清屏幕和精修素材 |

<Note>
  GPT-Image-2 当前按成功生成图片次数扣费。编辑图同样按成功返回的图片次数和尺寸档位计费。
</Note>
