Nano Banana 2 是 `gemini-3.1-flash-image-preview` 的产品展示名。它通过 DeepToken 暴露 Gemini 官方图像能力，使用 Gemini Native `generateContent` 协议，适合追求速度、成本和高分辨率平衡的图片生成与编辑场景。

## 接口

```http
POST https://YOUR_DOMAIN/v1beta/models/gemini-3.1-flash-image-preview:generateContent
```

## 能力速查

| 维度 | 说明 |
| --- | --- |
| 产品名 | Nano Banana 2 |
| 实际模型 ID | `gemini-3.1-flash-image-preview` |
| 协议 | Gemini Native REST |
| 生成端点 | `:generateContent` |
| 编辑端点 | 同 `:generateContent`，通过图片 part 传图 |
| 分辨率 | `512` / `1K` / `2K` / `4K` |
| 宽高比 | 常见比例和超宽/超长比例，具体可用值以官方能力为准 |
| 返回图片 | 推荐读取 `candidates[].content.parts[].fileData.fileUri` |

<Warning>
  Nano Banana 2 没有 OpenAI 风格 `/v1/images/edits`。请按 Gemini 原生多模态请求接入。
</Warning>

## 生产建议

<AccordionGroup>
  <Accordion title="优先用实际模型名" icon="tag">
    请求中使用 `gemini-3.1-flash-image-preview`。需要 4K 时通过 `generationConfig.imageConfig.imageSize` 指定，不要依赖非标准后缀模型名。
  </Accordion>
  <Accordion title="图片 part 与文本 part 分开" icon="braces">
    每个 `parts[]` 元素只放 `text`、`inlineData` 或 `fileData` 之一。不要在同一个 part 里同时放文本和图片。
  </Accordion>
  <Accordion title="输出优先读取 URL" icon="link">
    DeepToken 会优先把输出图片整理为 URL。生产解析时先找 `fileData.fileUri`，再兼容处理 `inlineData.data`。
  </Accordion>
  <Accordion title="设置足够长的超时" icon="timer">
    4K、高峰并发和复杂提示词会拉长响应时间。生产建议客户端超时至少 300 到 360 秒。
  </Accordion>
</AccordionGroup>
